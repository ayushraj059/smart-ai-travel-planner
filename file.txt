Prompt 5:
You are a senior AI engineer
Create a microservice endpoint inside of @backend with the name of rag.
The purpose of this pipeline is to create  a full end-to-end pipeline.
Firstly whatever data is there in dynamodb you have to convert it into embeddings using sentence transformers and store it in pinecone database for vector search.

- Create a function inside of @backend/rag/app/embeddings.py to convert the user details which are entered by users from @backend/user_details/app/main

- Now we are at the last stage of the rag pipeline in which the embeddings of the user query from @backend/rag/app/embeddings.py is compared with the vector embeddings of pinecone database retrieve the top-K most relevant places based on semantic similarity.
- The retrieved results are then passed to a reasoning layer powered by an open-source LLM Groq  to generate a structured, day-wise travel itinerary.
- The LLM processes both user constraints (budget, duration, preferences) and retrieved contextual data to perform intelligent decision-making, including filtering, ranking, and sequencing of locations while minimizing travel distance and ensuring feasibility.
- A post-processing validation layer is applied to enforce constraints such as time overlap, budget limits, and logical ordering, ensuring the final itinerary is realistic and executable.
- The final output is returned as a structured JSON response containing a detailed itinerary along with summary insights such as estimated cost and budget status.
